# Traffic Statistics (`sbx stats`)

sbx exposes sing-box's built-in Clash API (`experimental.clash_api`) so you
can monitor live traffic, active connections, memory use and per-user usage
directly from the command line or a third-party Web UI.

The API listens on **loopback only** (`127.0.0.1:9090`) and is protected by a
Bearer token generated once per install and stored at mode-600 inside
`/etc/sing-box/state.json`. It is **never** reachable from the public
internet.

---

## Quick start

```bash
sbx stats                  # live overview (throughput, total bytes, uptime)
sbx stats --json           # same data as JSON for scripts
sbx stats connections      # destination, user, up/down, age per connection
sbx stats users            # aggregated per-user breakdown
sbx stats disable          # turn off the Clash API
sbx stats enable           # turn it back on
```

`sbx stats` re-uses the multi-user metadata written by sing-box for the
Reality inbound, so `stats users` shows bytes-per-user without any extra
configuration.

### Example output

```

```

## Disabling the Clash API

At install time:

```bash
SBX_STATS_ENABLE=0 bash install.sh
```

At runtime:

```bash
sudo sbx stats disable   # writes .stats.enabled=false, regenerates config, restarts sing-box
sudo sbx stats enable    # reverses it
```

Disabling removes the `experimental.clash_api` block from `config.json`. The
secret is retained in state.json so re-enabling does not invalidate
bookmarked Web UI sessions.

### Rotating the secret

```bash
sudo jq 'del(.stats.secret)' /etc/sing-box/state.json | sudo tee /etc/sing-box/state.json.tmp >/dev/null
sudo mv /etc/sing-box/state.json.tmp /etc/sing-box/state.json
sudo sbx stats enable   # regenerates a new 64-char hex secret
```

## Connecting a Web UI (yacd / metacubexd)

The Clash API is not exposed publicly, so connect through an SSH tunnel:

```bash
# On your laptop:
ssh -L 9090:127.0.0.1:9090 user@your-server
```

Then, in the Web UI:

- **API Base URL**: `http://127.0.0.1:9090`
- **Secret**: the value of `.stats.secret` from `/etc/sing-box/state.json` on the server
  ```bash
  sudo jq -r .stats.secret /etc/sing-box/state.json
  ```

Recommended dashboards (both are open source, static-hosted, and speak the
Clash API protocol unchanged):

- [yacd](https://github.com/haishanh/yacd) — lightweight, stable
- [metacubexd](https://github.com/MetaCubeX/metacubexd) — richer features

sbx does not ship a built-in dashboard; relying on these third-party UIs
keeps the installer small and lets you pick whichever interface you prefer.

---

## Security notes

- The API always binds to `127.0.0.1`. If you need remote access, use SSH
  port-forwarding rather than changing the bind address.
- The secret is a 64-character hex string (256 bits of entropy) generated by
  `openssl rand -hex 32` on first install.
- State and config files are mode-600 and root-owned; only privileged users
  on the server can read the secret.
- Counters reset when sing-box restarts. Historical traffic is **not**
  persisted (there is no `cache_file` in the Clash API block).

## JSON schema (`sbx stats --json`)

```json
{
  "enabled": true,
  "uptime_seconds": 8043,
  "traffic": {
    "up_bps": 126976,
    "down_bps": 524288,
    "upload_total": 44433408,
    "download_total": 302874624
  },
  "connections": {
    "count": 37,
    "by_user": [
      { "user": "alice", "count": 22, "upload": 1234, "download": 5678 },
      { "user": "bob",   "count": 15, "upload":  456, "download":  789 }
    ]
  },
  "memory": { "inuse": 40060928, "oslimit": 0 }
}
```

When stats are disabled, the payload is just `{"enabled": false}`.
